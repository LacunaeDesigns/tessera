# v0.3 Occasions & Series Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking. Spec: [../features/occasions.md](../features/occasions.md).

**Goal:** Occasion library, prompt decks, milestone series wizard, the drawn letter ladder, co-written letters, and (conditionally) the grief set.

**Context for a fresh executor:** read `CLAUDE.md` first. The compose flow lives in `js/compose.js` (a step state machine rendering into `#compose-root` with an `el()` helper; sealing delegates to `TesseraExport.seal` — **the one seal path**, an architecture invariant). Occasions are data in `data/occasions.js` (`{ slug, title, coverLine, prompts[], canBeUndated? }`). Registry rendering: `js/registry.js` → `#registry-root`. Re-grep these interfaces before starting; this plan is written against the 2026-07-10 code.

**Ordering note:** Tasks are independent except Task 3 (wizard) which should land before Task 5 (co-written) touches the same compose code.

---

### Task 1: occasion library expansion (content, via /occasion)

- [ ] **Step 1:** Invoke the `/occasion` skill once per set. Sets to author (from the spec): milestone birthdays, graduation, first heartbreak, anniversaries, *to the next owner of this house*, *to a stranger in 100 years* (slug **`stranger-2126`** — this slug is load-bearing; the v2.x Hundred-Year Room selects on it), *to the person marrying my child*, retirement, last day of a career.
- [ ] **Step 2:** Each set: 3–4 prompts, cover line, bright-door rule holds across the shipped library. Data-only change to `data/occasions.js`.
- [ ] **Step 3:** `node tools/prose-check.js data/occasions.js` clean; read every prompt aloud; `node --check data/occasions.js`; chips appear in the browser. Commit per set or per sitting.

### Task 2: prompt decks (deck mode)

- [ ] **Step 1:** Data shape: add optional `deck: [{ q }]` (6–10 questions) to an occasion entry. Decks are interview-style; the last screen is always the stitch step, headed exactly: *"Now make it sound like you."*
- [ ] **Step 2:** `js/compose.js`: when the chosen occasion has a deck, the letter step (step 4) offers *"Answer a few questions instead"* → one question per screen (same `.compose-q` idiom), answers accumulate in `draft.deckAnswers[]`, autosaved like the letter body.
- [ ] **Step 3:** Stitch: concatenate answers into paragraph-separated draft text placed in the letter textarea for editing. A deck is **never** sealed unedited: the seal review shows a quiet note if `letterText === stitched(deckAnswers)` verbatim: *"These are still the questions' words. Make them yours first."* (soft warning, not a block).
- [ ] **Step 4:** Open question resolved here (leaning yes per spec): a checkbox at stitch time, default off, *"Keep the raw answers in the folder"* → adds `media/interview.txt` to the export file list and checksums. **That is a format touch:** `media/` is already legal per SPEC (verify with `/spec-sync`; if `media/` is not yet in SPEC §1, this becomes a format event — spec + schema + decisions.md same day).
- [ ] **Step 5:** Full /qa loop with a deck; Gate 2 over the deck strings; commit.

### Task 3: milestone series wizard

- [ ] **Step 1:** Entry: after the occasion step, occasions tagged `series: { label, defaultDates }` (new optional data field, e.g. child at 5/10/15/18) offer *"Write the whole ladder in one sitting."*
- [ ] **Step 2:** Wizard flow: shared `to`/`from`/custody collected once; then N letter steps, each with its own `openOn` (prefilled from `defaultDates`, editable) and its own letter text; a running sidebar lists the ladder so far. State: `draft.series[]` under the same `tessera_v1` draft key (extend `state.migrate()`).
- [ ] **Step 3:** Sealing a series = a loop over the existing single-letter seal (`TesseraExport.seal` per letter — the one-seal-path rule; no batch seal function). Produces N folders (N zip downloads or one combined download decision: default N separate zips, each letter stays a self-contained Tessera folder — a combined "family zip" is convenience only, decide with the author, log in decisions.md), N registry rows, and one print job: all kits concatenated into `#print-root` in date order.
- [ ] **Step 4:** /qa: the spec's fixture — a 3-letter series ⇒ 3 valid folders (each passes checksum verification), 3 registry rows, one print job with per-letter footers correct. Commit.

### Task 4: the letter ladder

- [ ] **Step 1:** `js/registry.js`: above the Waiting list, render an inline SVG timeline (no library): a horizontal hairline, "now" at the left margin, each waiting letter a small sealed-token mark (reuse a simplified 12px token ring) positioned by `openOn` distance, decade ticks labelled in words ("ten years", "twenty-five years"). Log-ish scale is acceptable for spread; whatever is chosen, label honestly.
- [ ] **Step 2:** Hover/tap a mark highlights its row (no other interactivity). `prefers-reduced-motion`: no animation at all.
- [ ] **Step 3:** Ladder prints with the register sheet, ink-light (hairline + outline marks only). 375px first: the ladder must not cause horizontal scroll — it scales, ticks thin out.
- [ ] **Step 4:** /qa + commit.

### Task 5: co-written letters

- [ ] **Step 1 (format decision first, its own commit):** two writers → `from` stays a single string in the manifest (*"Maria and Tomas"*) **or** becomes an array. Decision to take to `/spec-sync` + decisions.md: keep `from` a string (century-simple, human-readable) and add optional `writers: ["…","…"]`. Additive optional field = format event protocol (spec + schema + fixtures + decisions, same sitting).
- [ ] **Step 2:** Compose: step 3 gains *"We are writing this together"* → second `from` field; letter step offers alternating mode (labelled turns inserted as plain text) or merged mode (one voice).
- [ ] **Step 3:** Token: `/token-lab` decides the spec's open question — three-way break vs. two identical travel halves — on rendered visual evidence, within the determinism contract (a new art variant is a fixture birth for the new path, never a diff of existing fixtures). The seed derivation must remain exactly the SPEC §4–5 rules.
- [ ] **Step 4:** /qa full loop co-written; `/century-audit` row: a co-written folder still self-describes. Commit.

### Task 6: the grief set (conditional — may not ship, and that is a fine outcome)

- [ ] **Step 1:** Draft via `/occasion` with its full care: outside reading is **required** before shipping (the spec defers it here "or not at all").
- [ ] **Step 2:** At least one outside reader with lived experience reads the prompts; their sign-off (name/date, or "declined") is recorded in decisions.md. No sign-off, no ship — remove it from the release, not the rules.
