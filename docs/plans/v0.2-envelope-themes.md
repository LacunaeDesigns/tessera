# v0.2 Envelope & Print Themes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** An envelope fold-and-glue sheet and four print themes (letterpress · aerogramme · telegram · wax-seal), all within the print rules. Spec: [../features/print-kit.md](../features/print-kit.md) §v0.2; rules: [../design-language.md](../design-language.md).

**Architecture:** Pure print-CSS work plus one new sheet renderer in `js/print.js`. Themes are a `theme-<name>` class on `#print-root`, chosen in the print toolbar at print time; nothing enters the manifest (a reprint in another theme is the same letter). The instructions page never changes voice or layout.

**Context for a fresh executor:** read `CLAUDE.md`, then `js/print.js` (the `el()` idiom, `kitSheets(sealed)`, `sheet-footer` pattern) and the print engine section at the bottom of `style.css`. Before coding, re-grep the current sheet class names (`grep -n "el('section', 'sheet" js/print.js`) — this plan was written 2026-07-10 against `sheet-cover/letter/instructions/token/register`.

**Drift check before starting:** physical-walkthrough notes from the v0.1 release live in the release notes (PROGRESS.md log). If they flag margin/break problems, fix those in the base engine *first*, as their own commit.

### Task 1: envelope sheet

**Files:** Modify `js/print.js`, `style.css`; letter-facing copy → run `node tools/prose-check.js js/print.js` after.

- [ ] **Step 1:** Add `envelopeSheet(sealed)` to print.js beside the other sheet builders: a `section.sheet.sheet-envelope` containing a fold template sized to an A5-ish result (letter sheet folded in thirds fits): a rectangle laid out with dashed hairline fold lines (`.fold-line`), glue tabs (`.glue-tab`, hairline outline + the word "glue" in 8pt), and on the flap area the cover text: *A letter, sealed {written-in-words}.* / *For {to}.* / *Opens {open-date-in-words or "when the title says so"}.* Fold instructions in words under the template (scissors-free rule): numbered, plain, e.g. "1. Cut along the outer dashed line. 2. Fold the left and right thirds inward…".
- [ ] **Step 2:** CSS: `.sheet-envelope` geometry with mm units inside the print block; screen preview shows the same sheet as a card. Dashed hairlines only (`border: 0.5pt dashed`), no filled areas (ink-light rule).
- [ ] **Step 3:** Wire into the kit: envelope is offered, not forced — a toolbar checkbox "Include the envelope template" (default on for dated letters). Verify: A4 + Letter PDFs, template fits both with ≥15mm margins; cut-and-fold a real print once (author or anyone with a printer) before calling done.
- [ ] **Step 4:** Gate 2 prose scan; commit.

### Task 2: theme plumbing

**Files:** Modify `js/print.js` (toolbar select + class switch), `js/state.js` (remember last theme in `settings` via `migrate()`), `app.html` (toolbar select markup if the toolbar is static).

- [ ] **Step 1:** `<select id="print-theme">` in the print toolbar: letterpress (default) / aerogramme / telegram / wax-seal. On change: swap `theme-<name>` class on `#print-root`; persist choice to `settings.printTheme`.
- [ ] **Step 2:** Guard: the instructions sheet (`.sheet-instructions`) opts out — its rendering must be byte-identical across themes (assert visually; its CSS selectors must not be theme-scoped).
- [ ] **Step 3:** Commit before any theme styling exists (plumbing proven with letterpress = current look).

### Task 3: the three new themes (one commit each)

All themes restyle **within** the rules: ink-light, self-identifying footers, serif letter body, sizes in pt. Only cover + letter + token + envelope sheets respond.

- [ ] **aerogramme** — hairline red/blue dashed border pattern on the cover and envelope edges (border-image or repeating-linear-gradient at hairline weight; no filled bands), "PAR AVION"-style kicker in letterspaced 8pt caps, lighter rule weights.
- [ ] **telegram** — monospace date-stamped header line on the cover (`ui-monospace`; `SEALED 2026 JUL 10 · OPENS 2036 JUL 10 · TSR-…`), body stays serif (the letter is never monospace), double hairline rules.
- [ ] **wax-seal** — cover gains a circular seal *placement mark* (a dashed hairline circle + the words "seal here, if you have wax"), cover ornament is the letter's own token ring at 20mm. No printed solid "wax" — the mark invites the real thing.
- [ ] Per theme: A4 + Letter PDF check, ink-light check (no filled areas > 5% of page), Gate 2 prose scan on any new words, then commit.
