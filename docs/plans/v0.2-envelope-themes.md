# v0.2 Envelope & Print Themes — pre-plan brief

**Status:** brief, not yet an executable plan. Write the full plan when v0.1 has shipped and the print engine (style.css print layer) has survived a physical-printer walkthrough — theme work inherits whatever page-break/margin lessons that surfaces.

**Spec:** [../features/print-kit.md](../features/print-kit.md) §v0.2 additions; rules in [../design-language.md](../design-language.md).

## Scope
1. **Envelope generator** — one new sheet in `print.js`: fold-and-glue template sized to the letter sheet folded in thirds; dashed hairline fold/cut lines with instructions in words; cover text + dates printed on the flap. Pure print-CSS geometry; no engine changes.
2. **Print themes** — `letterpress` (default, current), `aerogramme`, `telegram`, `wax-seal`. Implementation direction: a `theme-<name>` class on the sheet container; themes restyle **within** the print rules (ink-light, self-identifying footers, serif body). The instructions page never changes voice or layout. Theme choice lives at print time (toolbar select), not in the format — a letter re-printed in a different theme is the same letter; nothing enters the manifest.

## Constraints to carry into the plan
- Ink-light rule holds per theme (aerogramme borders are hairline patterns, not filled bands; telegram header is monospace type, not a background).
- A4 + US Letter double preview per theme; the single-sided rule stays.
- Copy on the flap is letter-facing → Gate 2 prose scan.

## Definition of ready (to write the full plan)
- v0.1 shipped; physical walkthrough notes available.
- Decision logged on theme selector placement (print toolbar vs. seal screen).
